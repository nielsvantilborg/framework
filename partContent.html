partContent

Setup & Usage

  • partContent.css is included on default in the index.html document
  • Including magnificpopup-2.css is optional
<link rel='Stylesheet' type='text/css' href='css/partContent.css'> <link rel='Stylesheet' type='text/css' href='http://cdn.dailycms.com/css/magnificpopup-2.css'>

Javascript

  • jquery-1.12.2.js only needed for popup
  • jquerymagnificpopup-1.1.0.js is optional
<script type='text/javascript' src='http://cdn.dailycms.com/js/jquery-3.1.0.js'></script> <script type='text/javascript' src='http://cdn.dailycms.com/js/jquerymagnificpopup-1.1.0.js'></script>

partContentText

Note

  • It's possible to add an 'h1, h2, h3, etc' as first element within the 'partContent' container. Most common is 'h2'. This depends on what is best for SEO
  • It's allowed to apply an 'a' element with the property 'href' witin the h2. The 'a' can not be manually linked so there should be logic behind it. For example you apply the 'h2' in an overview and you link to a detail page. In that case it's almost always better and more clear to place the 'h2' outside the 'partContentText' but sometimes it's more efficient for CSS styling.
  • readMore iniside P: Optionally you can add a 'readMore' within the element 'p' at the end

partContentFileAdmin

Note

  • On every 'a' we generate an unique ID that can be tracked for analytics
  • It's possible to show the file name with or without file extension
  • It's possible to show the file size after the file name
  • Supports more filetypes than shown bellow

Demo

Heading 2 inside partContentFileAdmin

  • File name
  • File name with file extensions.pdf
  • File name with file size (123kb)
  • File name with a very long file name which can happen
  • File name
  • File name

HTML

<div class='partContentFileAdmin'> <h3>Heading 2 inside partContentFileAdmin</h3> <ul> <li><a class='partContentFileAdminPDF' href='#'>File name</a></li> <li><a class='partContentFileAdminDOC' href='#'>File name with file extensions.pdf</a></li> <li><a class='partContentFileAdminDOCX' href='#'>File name with file size (123kb)</a></li> </ul> </div> <div class='partContentFileAdmin'> <ul> <li><a class='partContentFileAdminXLS' href='#'>File name with a very long file name which can happen</a></li> <li><a class='partContentFileAdminXLSX' href='#'>File name</a></li> <li><a id='uniqueIdForTracking' class='partContentFileAdminZIP' href='#'>File name</a></li> </ul> </div>

partContentFileImageGallerySingle

Note

  • Can be used with or without <a href='#'></a> around the img
  • Optionally 'contentFileImageGalleryDescription' can be applied
  • Optionally you can show a higher resolution of the image in a popup
  • It's possible to use 'partContentFileImageGallery' with only one image without using 'partContentFileImageGallerySingle'

Demo

HTML

<div class='partContentFileImageGallery partContentFileImageGallerySingle'> <img src='http://pipsum.com/800x300.jpg?1' alt='Image description'> </div>

partContentFileImageGallery imagesInRow2

Notes

  • Can be used with or without ''
  • Can be sued with or without popup
  • Optionally you can show a higher resolution of the image in a popup
  • Optionally you can apply a more label on the latest images and hide the rest. Use contentFileImageContainerShowAmount for that
  • Class imagesInRow2, imagesInRow3, imagesInRow4 defines how many pictures will be shown in a row
  • partContentFileImageGalleryFooter is depreciated. You need a overflow:hidden on partContentFileImageGallery

partContentCode

Custom code can be applied within the div 'partContentCode'. For example an iFrame, a button or something else...

partContentCode note

  • Can be used for mulitple purposes like a iFrame or a button. The idea is that you can edit/modify this in the CMS

partContentCode code

<div class='partContentCode'> <a class='linkButton' href='#' title='Button name'>Button name</a> </div>
